### 大模型运行约定
当前项目后续的模型推理/服务接入方向已经明确：

- **最终大模型通过 vLLM 提供服务**

当前仓库里也已经有对应配置入口：
- 根目录 `.env.example` 中：
  - `VLLM__BASE_URL`
  - `VLLM__MODEL_NAME`
- 后端系统信息接口也会返回：
  - `vllm.base_url`
  - `vllm.model_name`

因此后续开发默认约定是：
- 不以本地直连模型文件为主
- 不以内嵌模型推理进程为主
- **以前后端通过 HTTP 调用 vLLM 服务为主**
- **接口兼容目标默认采用 vLLM 的 OpenAI ChatCompletion 协议**
- **`VLLM__BASE_URL` 约定指向 `/v1` 根路径，实际聊天补全请求走 `/chat/completions`**

下一位开发者在继续推进时，应把以下原则视为既定方向：
1. 模型配置优先通过根目录 `.env` 控制
2. Setup 页面应逐步把 vLLM 连通性展示得更清晰
3. 后续 AI 能力接入时，默认依赖 vLLM 提供标准接口
4. Justice UI / 后端在接入聊天能力时，优先按 OpenAI ChatCompletion 的 `model`、`messages` 等字段结构组织请求，避免自定义协议

- **前端不会自动拉起后端**

- Setup 页面只是检查状态并给出说明，不会启动服务

### 当前手动启动命令

#### 启动后端

#### 启动前端
